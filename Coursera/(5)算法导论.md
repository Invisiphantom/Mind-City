
# 分治算法

## 递归式

把原问题分解为$a$个子问题, 每个子问题规模为$1/b$
记原问题分解需要时间$D(n)$, 子问题合并需要时间$C(n)$
如果问题规模足够小, 那么只需要常量时间$\Theta(1)$
$T(n)=\begin{cases}
    \Theta(1) & n\leq c \\[0ex]
    aT(n/b)+D(n)+C(n) & n>c \end{cases}$



## 用主定理求解递归式

![image.png](https://s2.loli.net/2024/09/11/jLS2Q6uimfCgWwZ.png =500x)


主定理(Master Theorem)
对于递归式$T(n)=aT(n/b)+f(n)$
通过分析递归树得 $T(n)=\sum_0^{\log_b n}a^if(n/b^i)+O(n^{\log_b a})$
1. $f(n)\in O(n^{\log_ba-\epsilon})\implies T(n)\in\Theta(n^{\log_ba})$
2. $f(n)\in\Theta(n^{\log_ba})\implies T(n)\in\Theta(n^{\log_ba}\log n)$
3. $f(n)\in\Omega(n^{\log_ba+\epsilon})\land af(n/b)\leq cf(n)\implies T(n)\in\Theta(f(n))$

(1): $f(n)\in O(n^{\log_ba-\epsilon})\implies\sum_0^{\log_b n}a^if(n/b^i)\leq\sum_0^{\log_b n}a^if(n/b^i)$
$\begin{aligned}
    \sum_0^{\log_b n}a^if(n/b^i)=n^{\log_ba-\epsilon}\sum_0^{\log_bn}a^ib^{-i\log_ba}b^{i\epsilon}
\end{aligned}$



## 插入排序

```py
n = len(A)
for j in range(0, n):
    key = A[j] # 取出第j张牌, 将其插入前(j-1)有序牌堆中
    i = j - 1
    while i >= 0 and A[i] > key:
        A[i + 1] = A[i]
        i = i - 1
    A[i + 1] = key
```

### 可行性分析

循环不变量: 在每次循环开始时, 子数组$\{0,\cdots,j-1\}$是有序的

起始: 第0张牌是自有序的
递推: 已知第$\{0,\cdots,j-1\}$牌有序, 由算法过程可知, 插入第j张牌后仍保持有序
结束: 当j=n时, 满足第$\{0,\cdots,n-1\}$牌有序, 即整个牌堆有序

### 最坏时间复杂度分析

最坏情况: 如果输入数组已反向排序
那么对于第j张牌, while循环需要比较j次
因此时间复杂度 $\sum_1^{n-1} j=\frac{n(n-1)}{2}=\Theta(n^2)$

## 归并排序

def MERGE(A, lBin, rBin, rEnd):
    L = A[lBin:rBin]  # A[lBin, rBin)
    R = A[rBin:rEnd]  # A[rBin, rEnd)
    L.append(float("inf"))
    R.append(float("inf"))
    i = 0
    j = 0
    for k in range(lBin, rEnd):
        if L[i] <= R[j]:
            A[k] = L[i]
            i += 1
        else:
            A[k] = R[j]
            j += 1


def MERGE_SORT(A, bin, end):
    if bin < end - 1: # A[bin, end-1]
        mid = (bin + end) // 2
        MERGE_SORT(A, bin, mid)
        MERGE_SORT(A, mid, end)
        MERGE(A, bin, mid, end)

MERGE_SORT(A, 0, len(A))

```py
# 无极大哨兵版本
def MERGE(A, lBin, rBin, rEnd):
    L = A[lBin:rBin]  # A[lBin, rBin)
    R = A[rBin:rEnd]  # A[rBin, rEnd)
    i, j, k = 0, 0, lBin
    while i < len(L) and j < len(R):
        if L[i] <= R[j]:
            A[k] = L[i]
            i += 1
        else:
            A[k] = R[j]
            j += 1
        k += 1
    
    if i < len(L):
        A[k:rEnd] = L[i:]
    elif j < len(R):
        A[k:rEnd] = R[j:]
```


### 可行性分析

起始: 只含1个元素的组是有序的
递推: 已知`A[lBin, rBin)`和`A[rBin, rEnd)`是有序的
由算法过程可知, 合并后的`A[lBin, rEnd)`也是是有序的
结束: 最终得到的`A[0, len(A))`有序, 即整个数组有序


### 最坏时间复杂度分析

#### 归并排序分析

分解: 只需计算中间位置`mid`, 时间复杂度$\Theta(1)$
解决: 递归调用两个规模减半的子问题, 时间复杂度$2T(n/2)$
合并: 合并两个规模为$n/2$的有序数组, 时间复杂度$\Theta(n)$
$T(n)=\begin{cases}
    \Theta(1) & n=1 \\[0ex]
    2T(n/2)+\Theta(n) & n>1 \end{cases}$
