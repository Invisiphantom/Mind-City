
# 排序问题

输入: n个数的序列$A=\langle a_1,a_2,\cdots,a_n\rangle$
输出: 序列A的排序$A'=\langle a_1',a_2',\cdots,a_n'\rangle$, 满足$a_1'\leq a_2'\leq\cdots\leq a_n'$

```py
A = [5, 4, 3, 2, 1]
print(A) # 希望得到 [1, 2, 3, 4, 5]
```


## 插入排序

```py
n = len(A)
for j in range(0, n):
    key = A[j] # 取出第j张牌, 将其插入前(j-1)有序牌堆中
    i = j - 1
    while i >= 0 and A[i] > key:
        A[i + 1] = A[i]
        i = i - 1
    A[i + 1] = key
```

### 可行性分析

起始: 前1张牌是有序的
递推: 已知前j-1张牌有序, 由算法过程可知, 插入第j张牌后仍保持有序
结束: 当j=n+1, 此时前n张牌有序, 即整个牌堆有序

### 最坏时间复杂度分析

最坏情况: 如果输入数组已反向排序
那么对于第j张牌, while循环需要比较j次
因此时间复杂度 $\sum_2^n j=\frac{n(n+1)}{2}-1=\Theta(n^2)$

## 归并排序


```py
def MERGE(A, lBin, rBin, rEnd):
    L = A[lBin:rBin]  # A[lBin, rBin)
    R = A[rBin:rEnd]  # A[rBin, rEnd)
    L.append(float("inf"))
    R.append(float("inf"))
    i = 0
    j = 0
    for k in range(lBin, rEnd):
        if L[i] <= R[j]:
            A[k] = L[i]
            i += 1
        else:
            A[k] = R[j]
            j += 1


def MERGE_SORT(A, bin, end):
    if bin < end - 1: # A[bin, end-1]
        mid = (bin + end) // 2
        MERGE_SORT(A, bin, mid)
        MERGE_SORT(A, mid, end)
        MERGE(A, bin, mid, end)

MERGE_SORT(A, 0, len(A))
```

### 可行性分析

起始: 只含1个元素的组是有序的
递推: 已知`A[lBin, rBin)`和`A[rBin, rEnd)`是有序的
由算法过程可知, 合并后的`A[lBin, rEnd)`也是是有序的
结束: 最终得到的`A[0, len(A))`有序, 即整个数组有序

### 最坏时间复杂度分析

#### 分治算法

如果问题规模足够小$n\leq c$, 那么只需要常量时间$\Theta(1)$
假设把原问题分解为$a$个子问题, 每个子问题规模为$1/b$
记原问题分解需要时间$D(n)$, 子问题合并需要时间$C(n)$
$T(n)=\begin{cases}
    \Theta(1) & n\leq c \\[0ex]
    aT(n/b)+D(n)+C(n) & n>c \end{cases}$

#### 归并排序分析

分解: 只需计算中间位置`mid`, 时间复杂度$\Theta(1)$
解决: 递归调用两个规模减半的子问题, 时间复杂度$2T(n/2)$
合并: 合并两个规模为$n/2$的有序数组, 时间复杂度$\Theta(n)$
$T(n)=\begin{cases}
    \Theta(1) & n=1 \\[0ex]
    2T(n/2)+\Theta(n) & n>1 \end{cases}$
